## ❌ 常见错误日志

### 1. WiFi 连接失败

```
连接WiFi............
❌ WiFi连接失败！
```

**可能原因：**
- WiFi 名称或密码错误
- WiFi 信号太弱
- 路由器限制了连接

**解决方案：**
- 检查 `config.h` 中的 `WIFI_SSID` 和 `WIFI_PASS`
- 靠近路由器测试
- 重启路由器

---

### 2. 时间同步失败

```
同步时间中...
❌ 时间同步失败！
```

**可能原因：**
- DNS 解析失败
- NTP 服务器不可达

**解决方案：**
- 检查网络是否正常
- 尝试更换 NTP 服务器（代码中已配置阿里云和科大源）

---

### 3. 天气 API 请求失败

```
❌ 请求失败，响应码: 401
❌ 失败原因: Unauthorized
```

**可能原因：**
- API Key 错误或过期
- 请求次数超限

**解决方案：**
- 检查 `config.h` 中的 `SENI_KEY`
- 登录心知天气控制台检查配额

---

### 4. MQTT 连接失败

```
连接MQTT...MQTT失败,错误码: 5
```

**MQTT 错误码说明：**

| 错误码 | 说明 |
|--------|------|
| -4 | 连接超时 |
| -3 | 连接丢失 |
| -2 | 连接失败 |
| -1 | 服务器不可达 |
| 1 | 协议版本不正确 |
| 4 | 用户名或密码错误 |
| 5 | 未授权 |

**解决方案：**
- 检查 `config.h` 中的 MQTT 配置
- 确认 MQTT 服务器已启动
- 检查防火墙是否开放 1883 端口

## ⏱️ 性能时序分析

基于实测数据，单次完整运行耗时约 **43 秒**：

| 阶段 | 耗时 | 说明 |
|------|------|------|
| 上电初始化 | ~1.4s | Serial 初始化 |
| WiFi 连接 | ~5s | 连接路由器 |
| NTP 同步 | ~4s | 时间同步 |
| 天气 API | ~2s | 两次 HTTP 请求 |
| 传感器读取 | ~0.5s | SHT11 温湿度 |
| MQTT 上报 | ~0.5s | 发布数据 |
| 屏幕刷新 | ~33s | 全刷耗时最长 |
| **总计** | **~43s** | |

### 屏幕刷新时序

```
_PowerOn   : 108ms   // 上电
_refresh   : 33222ms // 刷新（最耗时）
_PowerOff  : 81ms    // 断电
```

> 💡 **优化建议**：墨水屏全刷耗时 33 秒是正常的，如需更快响应可考虑使用局部刷新（但要注意残影问题）。

## 🔋 休眠策略

### 白天模式（00:00 - 01:00 之外）

```c
SLEEP_TIME_DAY_US = 30 * 60 * 1000000ULL;  // 30 分钟
```

### 夜间模式（01:00 之后）

```c
SLEEP_TIME_NIGHT_US = 6 * 60 * 60 * 1000000ULL;  // 6 小时
```

> 夜间减少唤醒次数，节省电量。

## 📝 调试技巧

### 1. 快速测试不休眠

临时注释掉休眠代码，方便调试：

```c
// esp_sleep_enable_timer_wakeup(sleep_time_us);
// esp_deep_sleep_start();
```

### 2. 查看变量值

使用 `DEBUG_PRINTF` 输出：

```c
DEBUG_PRINTF("温度: %.2f, 湿度: %.2f\n", temp_c, humidity);
```

### 3. 测量代码执行时间

```c
uint64_t start = esp_timer_get_time();
// ... 你的代码 ...
uint64_t end = esp_timer_get_time();
DEBUG_PRINTF("耗时: %llu us\n", end - start);
```
